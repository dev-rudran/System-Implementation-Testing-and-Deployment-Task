In this practical task, you will move from **system design** to a working **Python-based software solution**. Your goal is to implement, test, and prepare for deployment an **AI-based** or **agent-based software system** that uses external **tools** during execution.

The task is focused on the real implementation phase of software development. You will not only write code, but also organize the project structure, document the system, test its behavior, and prepare it for practical use. The final result must demonstrate that your system is functional, understandable, testable, and ready for controlled deployment.

**Task idea:** You must develop a Python system in which an **AI component**, **intelligent agent**, or **multi-agent workflow** solves a practical problem by using one or more **tools**. A tool can be, for example, a calculator function, file reader, web data retrieval module, search module, database access function, API connector, or another external utility that the agent can call to complete the task.

### Task Requirement

You must create a Python project that implements an **AI-assisted software solution** or an **agent-based software system**. The system must receive an input from the user, process the request, use at least one tool during problem solving, and return a meaningful result.

Your system may be implemented as:

*   a **single intelligent agent** that uses tools to complete tasks,
*   a **multi-agent system** in which different agents perform different responsibilities,
*   an **AI-supported assistant** that uses tools to retrieve, calculate, validate, transform, or generate information.

Examples of suitable project directions:

*   an agent that analyses input files and generates structured conclusions,
*   an AI study assistant that searches information and prepares answers using tools,
*   an agent workflow that validates code, selects suitable data structures, and proposes improvements,
*   a software assistant that uses APIs, local files, or search tools to solve a user-defined task.

### What your system must include

*   **Python implementation** with clear project structure,
*   **AI-based or agent-based logic**,
*   **tool usage** during execution,
*   **input and output handling**,
*   **testing** of the main functionality,
*   **deployment preparation**,
*   **documentation** for both developer and user.

### Implementation Phase

During implementation, you must transform your previously designed concept into a working Python system. The code should be divided into meaningful modules and written in a structured way. Each main part of the system must have a clear purpose.

In this phase, you should:

*   implement the main program logic,
*   create the agent or AI workflow,
*   define and connect the tools used by the system,
*   handle user input and output,
*   document important classes, functions, and modules,
*   prepare configuration files, dependencies, and startup instructions.

### Testing Phase

Testing must be performed **together with implementation**. You should not wait until the entire system is finished. As you develop the software, you must verify that each component works correctly.

Your testing should include:

*   **functional testing** of the main workflow,
*   **tool testing** to verify that the connected tools work correctly,
*   **input validation testing**,
*   **error handling testing**,
*   **at least several test scenarios** with expected results.

You may use simple Python test scripts, **unittest**, **pytest**, or your own structured validation approach. The important part is that the testing process is visible, meaningful, and connected to the implemented functionality.

### Versioning

You must use **Git** and store your project in a **GitHub repository**. Version control is part of the task and must demonstrate how your software evolved during development.

Your repository should include:

*   regular commits with meaningful commit messages,
*   a clear folder structure,
*   source code, tests, documentation, and support files,
*   a README file in English.

### Deployment Preparation

You do not need to publish a large production system online, but you must prepare your software so that another user can run it in a controlled way. This means the system must be installable and executable using the instructions you provide.

Your deployment preparation should include:

*   dependency list such as **requirements.txt**,
*   startup instructions,
*   configuration explanation,
*   environment variable description if needed,
*   clear explanation of how the user launches and uses the program.

### Data Porting and Conversion

If your system uses external data, files, prompts, structured records, or API responses, you must explain how the data is transformed, cleaned, converted, or passed between components. This is especially important if one tool returns data in one format and another part of the system requires another format.

You should show:

*   what input format is used,
*   how the data is converted or adapted,
*   how your system preserves correctness and consistency of the data.

### System Deployment Strategy

In your report, briefly describe what deployment strategy would be suitable for your solution. You are not required to fully implement advanced deployment models, but you must show that you understand how such a system could be released safely.

For example, you may explain whether your system would be deployed as:

*   a local application,
*   a command-line tool,
*   a web service,
*   an API-based assistant,
*   a staged release with testing before production use.
